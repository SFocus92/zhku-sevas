# 📦 Сборка в EXE — Полное руководство

## ⚠️ Важное замечание

**Next.js 16 невозможно полностью упаковать в один EXE файл** из-за:
- Динамической компиляции (Turbopack)
- Серверных компонентов React (RSC)
- Сложной системы роутинга App Router

Однако есть несколько рабочих альтернатив!

---

## ✅ Доступные варианты

### Вариант 1: Портативная папка (⭐ РЕКОМЕНДУЕТСЯ)

Создаётся папка `portable/` со всем необходимым для работы:

```
portable/
├── ЗАПУСТИТЬ.bat       # Скрипт запуска
├── standalone/         # Production сборка Next.js
├── db/                 # База данных SQLite
├── prisma/             # Prisma схема
├── .env                # Конфигурация
└── package.json        # Зависимости
```

**Как собрать:**

```bash
BUILD_EXE.bat
# Выберите: 1
```

Или напрямую:
```bash
СОЗДАТЬ_PORTABLE.bat
```

**Преимущества:**
- ✅ Работает сразу после копирования
- ✅ Все файлы в одной папке
- ✅ База данных включена
- ✅ Легко обновлять (заменить папку `standalone`)
- ✅ Доступ с других устройств сохраняется
- ✅ Не требует Python для сборки

**Требования:**
- Node.js 18+ должен быть установлен

**Как использовать:**
1. Скопируйте папку `portable` в нужное место
2. Запустите `ЗАПУСТИТЬ.bat`
3. Приложение откроется в браузере

### Вариант 2: EXE Лаунчер

Создаётся EXE-файл, который управляет запуском:

**Как собрать:**

```bash
BUILD_EXE.bat
# Выберите: 2
```

Или напрямую:
```bash
СБОРКА_ЛАУНЧЕРА.bat
```

**Требования:**
- Python 3.8+ для сборки
- Node.js должен быть установлен у пользователя

**Преимущества:**
- ✅ Один EXE файл
- ✅ Выглядит как настоящее приложение
- ✅ Автоматически открывает браузер

**Недостатки:**
- ❌ Требует установленный Node.js
- ❌ Нужен Python для сборки
- ❌ Все равно нужны папки с проектом рядом

---

### Вариант 3: Electron (полноценное Desktop приложение)

Создаётся приложение с собственным окном браузера:

**Установка:**
```bash
npm install electron electron-builder --save-dev
```

**Сборка:**
```bash
npm run electron:build
```

**Преимущества:**
- ✅ Полноценное окно приложения
- ✅ Можно упаковать всё в один EXE
- ✅ Работает без Node.js у пользователя

**Недостатки:**
- ❌ Большой размер (~150 МБ)
- ❌ Сложная настройка с Next.js
- ❌ Требуется дополнительная конфигурация

---

### Вариант 4: Tauri (современная альтернатива)

Лёгкое Desktop приложение на Rust + системный WebView:

**Требования:**
- Rust и Cargo
- Node.js

**Установка:**
```bash
npm create tauri-app@latest
```

**Преимущества:**
- ✅ Маленький размер (~5 МБ)
- ✅ Высокая производительность
- ✅ Нативный WebView системы

**Недостатки:**
- ❌ Требует Rust для сборки
- ❌ Сложная миграция с Next.js

## 📋 Подробная инструкция: Портативная версия

### Шаг 1: Сборка

```bash
# Запустите скрипт
BUILD_EXE.bat

# Или напрямую
СОЗДАТЬ_PORTABLE.bat
```

### Шаг 2: Проверка

После сборки будет создана папка:
```
portable/
├── ЗАПУСТИТЬ.bat
├── standalone/
│   ├── .next/
│   ├── node_modules/
│   ├── db/
│   ├── server.js
│   └── .env
├── db/
└── prisma/
```

### Шаг 3: Запуск

1. Перейдите в папку `portable`
2. Запустите `ЗАПУСТИТЬ.bat`
3. Сервер запустится на порту 3000
4. Браузер откроется автоматически

### Шаг 4: Копирование

Скопируйте папку `portable`:
- На флешку
- В другую папку
- На другой компьютер (с установленным Node.js)

## 🔧 Решение проблем

### Ошибка: "Node.js не найден"

Установите Node.js с https://nodejs.org/
Рекомендуется версия **LTS (20.x или выше)**

### Ошибка: "Prisma Client not generated"

Запустите в папке проекта:
```bash
npx prisma generate
```

### Ошибка: "Port 3000 is already in use"

Остановите другие процессы Node.js:
```bash
taskkill /F /IM node.exe
```

Или измените порт в `.env`:
```env
PORT=3001
```

### Ошибка при сборке лаунчера: "Python not found"

Установите Python 3.8+ с https://www.python.org/

**Важно:** При установке отметьте галочку **"Add Python to PATH"**

## 📝 Для разработчиков

### Структура portable версии

```
portable/
├── ЗАПУСТИТЬ.bat         # Главный скрипт
├── standalone/           # Next.js standalone сборка
│   ├── .next/           # Скомпилированные файлы
│   ├── node_modules/    # Зависимости
│   ├── db/              # База данных
│   ├── server.js        # Точка входа
│   └── .env             # Конфигурация
├── db/                   # Резервная копия БД
├── prisma/               # Prisma схема (для миграций)
└── package.json          # Для установки зависимостей
```

### Обновление portable версии

1. Внесите изменения в основной проект
2. Запустите `BUILD_EXE.bat` снова
3. Папка `portable` будет пересоздана

### Изменение порта

В файле `portable/.env`:
```env
DATABASE_URL="file:./db/database.db"
PORT=3001
```
