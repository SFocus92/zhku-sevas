# ✅ АУТЕНТИФИКАЦИЯ ДОБАВЛЕНА — ЧТО СДЕЛАНО

## 📁 Созданные файлы

| Файл | Назначение |
|------|------------|
| `src/middleware.ts` | Проверка авторизации перед входом |
| `src/app/login/page.tsx` | Страница входа (пароль) |
| `src/app/api/auth/route.ts` | API проверки пароля |
| `src/app/api/logout/route.ts` | API выхода |
| `src/components/logout-button.tsx` | Кнопка выхода |
| `src/components/header.tsx` | Универсальный хедер |

---

## 🔐 Как это работает

```
1. Клиент заходит на сайт
   ↓
2. Middleware проверяет куку auth_token
   ↓
3. Если куки нет → редирект на /login
   ↓
4. Клиент вводит пароль
   ↓
5. Пароль проверяется через API
   ↓
6. Если верно → устанавливается кука на 30 дней
   ↓
7. Клиент получает доступ к сайту
```

## ⚙️ НАСТРОЙКА (2 шага)

### ШАГ 1: Локально

1. Откройте файл `.env`
2. Найдите строку: `AUTH_PASSWORD="ваш_пароль_здесь"`
3. Замените на ваш пароль, например:
   ```env
   AUTH_PASSWORD="ZhkuSevastopol2026!"
   ```
4. Сохраните
5. Перезапустите: `ЗАПУСТИТЬ.bat`

### ШАГ 2: На Vercel

1. Откройте https://vercel.com/dashboard
2. Выберите проект `zhku-sevastopol`
3. Перейдите: **Settings** → **Environment Variables**
4. Нажмите **Add New**
5. Заполните:
   - **Name:** `AUTH_PASSWORD`
   - **Value:** ваш пароль (тот же что локально)
   - **Environment:** ✅ Production, ✅ Preview, ✅ Development
6. Нажмите **Save**
7. Сделайте **Redeploy**:
   - **Deployments** → последний деплой → **⋮** → **Redeploy**

## 🐛 Возможные проблемы

### Страница входа не появляется

**Причина:** Middleware не срабатывает

**Решение:**
1. Проверьте что `src/middleware.ts` существует
2. Проверьте что `AUTH_PASSWORD` установлен в `.env`
3. Перезапустите сервер

---

### Ошибка "Сервер не настроен"

**Причина:** `AUTH_PASSWORD` не установлен на Vercel

**Решение:**
1. Добавьте `AUTH_PASSWORD` в Vercel Environment Variables
2. Сделайте Redeploy

---

### Выход не работает

**Причина:** Кука не удаляется

**Решение:**
1. Очистите куки в браузере (Ctrl+Shift+Delete)
2. Попробуйте снова
