# 🏠 ЖКУ СЕВАСТОПОЛЬ - Инструкция по запуску

## ✅ Быстрый старт (рекомендуется)

### Вариант 1: Один компьютер
1. Запустите **ЗАПУСТИТЬ.bat**
2. Сервер автоматически соберётся и откроет браузер
3. Готово!

### Вариант 2: Доступ с других устройств
1. Запустите **START.bat** → выберите режим 3 (Quick Start)
2. Запомните IP адрес из окна запуска (например, `192.168.1.100`)
3. На телефоне/планшете откройте браузер
4. Введите: `http://192.168.1.100:3000`

---

## 📱 Подключение с других устройств

### Автоматическое определение IP:
При запуске вы увидите:
```
Local:   http://localhost:3000
Network: http://192.168.1.100:3000
```

### Что нужно:
- ✅ Все устройства должны быть в **одной сети** (один Wi-Fi роутер)
- ✅ Брандмауэр Windows должен разрешать порт 3000

### Если не подключается:
1. **Проверьте брандмауэр:**
   - Панель управления → Брандмауэр Защитника Windows
   - Дополнительные параметры
   - Правила для входящих подключений → Создать правило
   - Для порта 3000 (TCP)

2. **Или временно отключите брандмауэр** для проверки

## 🚀 Создание EXE файла

### Автоматическая сборка:
1. Запустите **BUILD_EXE.bat**
2. Дождитесь окончания сборки
3. Получите файл `zhku_sevastopol.exe`

### Ручная сборка:
```bash
# Сборка проекта
npm run build

# Установка pkg
npm install -g pkg

# Создание EXE
pkg standalone/server.js --targets node18-win-x64 --output zhku_sevastopol.exe
```

## ❓ Частые вопросы

### Q: Как остановить сервер?
**A:** Закройте чёрное окно (консоль) где запущен сервер

### Q: Где мои счета?
**A:** В папке `db/database.db` - не удаляйте эту папку!

### Q: Можно ли запустить на телефоне?
**A:** Нет, но можно **открыть в браузере телефона** если компьютер включен

### Q: Как перенести на другой компьютер?
**A:** Скопируйте **всю папку** проекта + установите Node.js

### Q: Нужен ли интернет для работы?
**A:** Нет, всё работает локально. Интернет только для initial setup

## 📞 Поддержка

Если что-то не работает:
1. Проверьте что Node.js установлен: `node --version`
2. Очистите кэш: удалите папку `.next`
3. Пересоберите: `npm run build`
4. Перезапустите сервер
