# 🔐 ИСПРАВЛЕНИЕ АУТЕНТИФИКАЦИИ НА VERCEL

## ❌ Проблема
Сайт пропускает без пароля, потому что middleware не проверял пароль правильно.

## ✅ Что исправлено

1. **Middleware** — теперь проверяет только наличие куки (не использует process.env)
2. **API /api/auth** — правильно устанавливает куку с флагоми для Vercel
3. **Защита** — теперь нельзя обойти через URL

### Шаг 2: Проверьте AUTH_PASSWORD на Vercel

1. Откройте https://vercel.com/dashboard
2. Выберите проект `zhku-sevas`
3. Перейдите: **Settings** → **Environment Variables**
4. Проверьте что есть переменная:
   - **Name:** `AUTH_PASSWORD`
   - **Value:** ваш пароль (например `Maksim2026`)
   - **Environment:** ✅ Production, ✅ Preview, ✅ Development

Если нет — добавьте!

---

### Шаг 3: Сделайте Redeploy

1. Перейдите в **Deployments**
2. Кликните на последний деплой
3. Нажмите **⋮** (три точки) → **Redeploy**
4. Дождитесь завершения

## 🧪 Проверка

### 1. Откройте сайт
```
https://zhku-sevas.vercel.app
```

### 2. Должна появиться страница входа
- Логотип
- Поле ввода пароля
- Кнопка "Войти"

### 3. Введите пароль
- Введите ваш пароль (например `Maksim2026`)
- Нажмите **Войти**

### 4. Проверьте защиту
- Сайт открылся ✅
- Теперь попробуйте выйти (кнопка справа вверху)
- После выхода снова откройте `https://zhku-sevas.vercel.app`

### 5. Проверьте обход через URL
- После выхода попробуйте открыть `https://zhku-sevas.vercel.app/` (без /login)
- **Должна перекинуть на /login!** ✅

## ⚠️ Если всё ещё пропускает без пароля

### Причина 1: Кука осталась в браузере

**Решение:**
1. Откройте DevTools (F12)
2. Перейдите в **Application** → **Cookies**
3. Удалите все куки для вашего домена
4. Обновите страницу

Или просто:
- Откройте сайт в режиме инкогнито (Ctrl+Shift+N)
- Должна появиться страница входа

---

### Причина 2: AUTH_PASSWORD не установлен на Vercel

**Решение:**
1. Проверьте Settings → Environment Variables
2. Добавьте `AUTH_PASSWORD` если нет
3. Сделайте Redeploy

---

### Причина 3: Кэширование на Vercel

**Решение:**
1. Deployments → последний деплой
2. ⋮ → **Redeploy** (не просто Retry, а именно Redeploy)

---

## 🔍 Как это работает теперь

```
1. Клиент заходит на сайт
   ↓
2. Middleware проверяет куку auth_token
   ↓
3. Если куки нет → редирект на /login
   ↓
4. Если кука есть → доступ к сайту
   ↓
5. Кнопка "Выйти" удаляет куку → редирект на /login
```

**Теперь нельзя обойти через URL — middleware всегда проверяет куку!**
