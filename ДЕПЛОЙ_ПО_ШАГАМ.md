# 🎯 ДЕПЛОЙ ПО ШАГАМ — НАЖИМАЙ ПО ПОРЯДКУ!

## 📌 ЭТАП 1: Создание базы данных (5 минут)

### Шаг 1.1: Открываем Neon
```
👉 https://neon.tech
```
- Нажмите **Sign Up**
- Выберите **Continue with GitHub**
- Войдите через ваш GitHub аккаунт

### Шаг 1.2: Создаём проект
- Нажмите **Create a project**
- Название: `zhku-sevastopol`
- Нажмите **Create**

### Шаг 1.3: Копируем DATABASE_URL
- **Скопируйте Connection String** (появится сразу после создания)
- Выглядит так: `postgresql://user:password@ep-xxx-xxx.region.aws.neon.tech/neondb?sslmode=require`
- **Сохраните в блокнот!**

---

## 📌 ЭТАП 2: Настройка локально (3 минуты)

### Шаг 2.1: Создаём .env.local
1. Откройте папку проекта: `d:\NEW_PROGECT\zhku_sevastopol_MAKSIM`
2. Скопируйте файл `.env.example`
3. Вставьте как `.env.local`
4. Откройте `.env.local` и замените строку DATABASE_URL на вашу из Neon

### Шаг 2.2: Применяем базу данных
Запустите файл (дважды кликните):
```
ПРИМЕНИТЬ_БАЗУ.bat
```

Дождитесь сообщения «ГОТОВО!»

### Шаг 2.3: Проверяем работу
Запустите:
```
ЗАПУСТИТЬ.bat
```
Откройте http://localhost:3000 — всё должно работать!

## 📌 ЭТАП 3: Загрузка на GitHub (5 минут)

### Шаг 3.1: Подготовка Git
Запустите файл (дважды кликните):
```
ПОДГОТОВИТЬ_GIT.bat
```

### Шаг 3.2: Создаём репозиторий на GitHub
1. Откройте https://github.com/new
2. Имя репозитория: `zhku-sevastopol`
3. Выберите **Private** (рекомендуется)
4. Нажмите **Create repository**

### Шаг 3.3: Отправляем код
Вернитесь в консоль после Шага 3.1 и выполните команды:

```powershell
# Замените ВАШ_НИК на ваш username GitHub
git remote add origin https://github.com/ВАШ_НИК/zhku-sevastopol.git

# Отправка
git push -u origin main
```

## 📌 ЭТАП 4: Подключение Vercel (3 минуты)

### Шаг 4.1: Импорт проекта
1. Откройте https://vercel.com/new
2. Войдите через GitHub
3. Найдите репозиторий `zhku-sevastopol`
4. Нажмите **Import**

### Шаг 4.2: Добавляем DATABASE_URL
**⚠️ ВАЖНО: Сделайте это ДО нажатия Deploy!**

1. Раскройте **Environment Variables**
2. Нажмите **Add New**
3. Заполните:
   - **Name:** `DATABASE_URL`
   - **Value:** ваша строка из Neon (Шаг 1.3)
   - **Environment:** ✅ Production, ✅ Preview, ✅ Development
4. Нажмите **Save**

### Шаг 4.3: Деплой
- Нажмите **Deploy**
- Дождитесь (1-2 минуты)
- Нажмите **Continue to Dashboard**

## 🆘 Если что-то пошло не так

| Проблема | Решение |
|----------|---------|
| Ошибка в Шаге 2.2 | Проверьте DATABASE_URL в .env.local |
| Не пускает на GitHub | Проверьте логин/пароль GitHub |
| Ошибка при деплое | Проверьте DATABASE_URL в Environment Variables Vercel |
| Пустая база | Запустите `npm run db:push` локально |

### Логи ошибок Vercel:
1. Dashboard → **Deployments**
2. Кликните на деплой
3. **View Logs**
